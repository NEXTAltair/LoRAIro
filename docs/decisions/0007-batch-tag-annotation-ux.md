# ADR 0007: Batch Tag Annotation UX

- **日付**: 2026-02-09
- **ステータス**: Accepted

## Context

バッチタグアノテーション UI で以下の問題があった:
1. ステージング一覧（左50%）とアノテーション操作（右50%）が水平分断されており、ユーザーが左右を交互に確認する必要がある
2. ステージング画像数が静的テキスト表示のみ（0件でも実行ボタンが有効）
3. アノテーション進捗フィードバックが弱い（どの画像？どのモデル？が UI に非表示）

## Decision

- ステージング画像数を動的カウント表示: "◎ ステージング: 25枚 / 500枚"
- 0件時は実行ボタンを Disabled に
- `WorkerProgress` Signal を UI に接続して現在処理中の画像・モデルを表示
- レイアウト改善: ステージング一覧とアノテーション操作の視覚的統合

## Rationale

UX 改善により誤操作（0件でのアノテーション実行）を防止。`WorkerProgress` Signal は既存実装があるため接続コストが低い。

## Consequences

- `labelAnnotationTarget` が動的更新される（静的テキストから動的表示へ）
- `btnAnnotationExecute` がステージング件数に連動して Enable/Disable
- ステージング件数変更時のシグナル接続が必要
