# ADR 0020: CLI Message Language Policy

- **日付**: 2026-04-27
- **ステータス**: Accepted

## Context

`lorairo-cli` のユーザー向けメッセージが日本語と英語で混在していた（ISSUE #203）。
コマンドによって言語が異なり、一貫性がなかった。

## Decision

**詳細な説明を含む ERROR/WARNING メッセージは英日併記とする。**

- 英語を第1行、日本語を第2行に表示する
- プログレスバーラベル・テーブルヘッダー・省略表示（`... and N more`）・ヒント例など短い要素は英語のみで可
- `loguru logger` の内部ログ（`logger.warning/debug/error`）は日本語のままで可

## Rationale

- **英語のみ統一** は変更コストが低いが、主要ユーザー（日本語話者）への配慮がない
- **日本語のみ統一** は CLI ツールの業界慣例から外れ、変更コストが大きい
- **英日併記** はどちらのユーザーも読めるメッセージを実現しつつ、最小限の変更で一貫性を確保できる
- i18n（gettext等）は現フェーズでは YAGNI

## Consequences

- ERROR/WARNING メッセージは2行になり若干長くなるが、許容範囲
- 新たにユーザー向けメッセージを追加する際は本 ADR のルールに従う
- テストでメッセージ内容をアサートする場合は英語部分を使用する（安定性のため）
