# ADR 0006: Pagination Approach

- **日付**: 2026-02-05
- **ステータス**: Accepted

## Context

18,252 件の検索結果表示でタイムアウト発生。QGraphicsScene が全件の QPixmap をメモリに保持（非圧縮換算 ~18GB）していた。LIMIT/OFFSET なしの全件取得が根本原因。

## Decision

**従来型ページネーション（ページ番号ナビゲーション）**を採用:
- DB クエリに `LIMIT/OFFSET` 追加（100件/ページ）
- ページ切り替え時に新規クエリ実行
- UI 下部にページネーションバー追加

## Rationale

検討した4案の評価:

| 案 | 実装難易度 | メモリ効率 | UX |
|----|-----------|-----------|-----|
| 従来型ページネーション ★ | 低 | 高 | 中 |
| 仮想スクロール | 高 | 高 | 高 |
| 無限スクロール | 中 | 中 | 中 |
| サムネイルサイズ削減 | 低 | 中 | 低 |

仮想スクロールは UX 最良だが QGraphicsScene の大幅書き換えが必要。現時点では実装コストと緊急度のバランスで最シンプルな従来型を選択（YAGNI）。

## Consequences

- メモリ使用量: 全件 → 1ページ分（100件）に削減
- ページ遷移時の待機時間が発生（トレードオフ）
- 複数選択がページをまたぐと複雑になる可能性
- 将来的に仮想スクロールへ移行する際は LIMIT/OFFSET 基盤をそのまま活用可能
